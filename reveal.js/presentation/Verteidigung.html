Anforderungen

  • Erstellung von Raumbuchungen/Terminen
  • Garantie der Überschneidungsfreiheit von Terminen

Frontend - Views

  • Raumpläne
  • Belegungsübersicht
  • Liste der Reservierungen
  • Buchen, Modifizieren und Löschen von Buchungen

Frontend - Architektur

  • einheitliche Schnittstelle zum Backend
  • einzelne Eventhandler für Views
  • asynchrone Funktionsweise

Frontend - Planansicht

  • einmaliges Laden der Gebäudedaten
  • dynamisches Nachladen der Pläne(nochmal nachgucken!)
  • Status(aktuelle Auswahl)

Frontend - Belegungsansicht

  • abhängig von der Auswahl in der Planansicht
  • Anzeige der Daten aus MVS
  • Zeitstrahlartige Anzeige der Raumbelegung

Frontend - Reservierungsansicht

  • mehrere Unteransichten
  • globale Übersicht der Reservierungen
  • Untermenüs durch überlagernde Modals realisiert
  • in Reservierungsmenü eingebettete Planansicht
  • Anzeige konkurrierender Veranstaltungen
  • farbliche Codierung von Konflikthäufigkeiten
